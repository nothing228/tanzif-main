# tanzif — территория роскошной чистоты

Лендинг цифровой экосистемы премиальной химчистки TANZIF (Ташкент): Vite + React + TypeScript + SCSS.

Ориентиры UX/позиционирования: Rinse (простота заказа), Mulberrys (премиальность), Embassy Cleaners (блоки доверия), The Clean Wash (конверсионные CTA), Orchid Cleaners (структура контента), UClean (готовность к сети) — реализовано в собственном стиле TANZIF.

## Структура сайта

Весь сайт — одна страница; отдельными маршрутами вынесены только вход и профиль.

| Маршрут | Содержание |
|---|---|
| `/` | Одностраничник: герой → «Как это работает» → разделы экосистемы → прайс-лист → курьер → трекинг → камеры → до/после → AI-консьерж → архив → бизнес → клуб → эко → знания → доверие |
| `/auth` | Вход / регистрация |
| `/profile` | Профиль: карта участника, бонусы, последний заказ, настройки, выход |

Меню в шапке — якорные ссылки с плавной прокруткой и подсветкой текущей секции ([useScrollSpy](src/hooks/useScrollSpy.ts)). Со страницы профиля переход к секции возвращает на главную и сразу встаёт на нужное место ([AnchorLink](src/components/AnchorLink.tsx)).

## Авторизация

Сайт открыт для просмотра без входа. Кнопка **«Вход / Регистрация»** — в шапке слева от переключателей.
Окно входа всплывает только при целевом действии: **оформление заказа**, **вызов курьера**, **отправка сообщения AI-консьержу**.

- Вход и регистрация **только по узбекскому номеру** (+998 и 9 цифр, префикс фиксирован)
- Пароль **минимум 8 символов**, при регистрации — подтверждение
- **Один номер нельзя зарегистрировать дважды**
- Пароли хранятся как SHA-256-хеш, сессия сохраняется между визитами
- ⚠️ Демо-режим: аккаунты хранятся в localStorage этого браузера. Для входа «с любого устройства» нужен сервер — слой [src/auth/AuthContext.tsx](src/auth/AuthContext.tsx) имеет async-API (register/login/logout) и заменяется на HTTP-бэкенд без правок остального кода.

## AI-консьерж

Консьерж работает через модель Claude. Ключ хранится **только на сервере** — в
Netlify-функции [netlify/functions/concierge.ts](netlify/functions/concierge.ts); браузер
ходит в неё, а не в API напрямую.

> ⚠️ Ключ нельзя объявлять как `VITE_ANTHROPIC_API_KEY`. Всё, что начинается с `VITE_`,
> вкомпилируется в JS-бандл и станет доступно каждому посетителю сайта.

**Настройка на проде:** Netlify → Site configuration → Environment variables → переменная
`ANTHROPIC_API_KEY`. Больше ничего менять не нужно, `netlify.toml` уже настроен.

**Локальная разработка:** `npm run dev` поднимает только фронтенд, без функций — консьерж
при этом молча падает на офлайн-логику по ключевым словам. Чтобы проверить именно ИИ:

```bash
npm i -g netlify-cli
cp .env.example .env    # вписать реальный ключ
npm run dev:ai          # vite + функции на http://localhost:8888
```

**Как устроены ответы.** Модель не знает цен наизусть — чтобы назвать любую сумму, она
обязана вызвать инструмент `search_pricelist`, который ищет по реальному прайсу
([src/data/priceSearch.ts](src/data/priceSearch.ts)). Придумать цену она физически не может.
Прайс на русском, поэтому модель сама переводит слово клиента («palto» → «пальто»);
таблица синонимов в `priceSearch.ts` дочищает случаи, где прайс называет вещь иначе
(«ковёр» → «палас/ковролин/турецкий»).

Если функция недоступна или ключ не задан, чат откатывается на прежний движок по ключевым
словам из [src/data/concierge.ts](src/data/concierge.ts) — виджет продолжает отвечать на
типовые вопросы о ценах и пятнах.

## Прайс-лист и AI-консьерж

- [src/data/pricelist.ts](src/data/pricelist.ts) — сгенерирован из официальной таблицы (197 услуг, 38 групп, артикулы и единицы измерения). Чтобы обновить цены, выгрузите лист в CSV и перегенерируйте файл.
- [src/components/SpotlightCard.tsx](src/components/SpotlightCard.tsx) — золотая подсветка карточек за курсором (React Bits). Полиморфный (`as`), поэтому применяется к существующим карточкам без изменения их вёрстки: разделы экосистемы, блоки доверия, знания, B2B.
- [src/components/StaggeredMenu.tsx](src/components/StaggeredMenu.tsx) — мобильное меню на компоненте StaggeredMenu (React Bits) на GSAP: золотые подложки выезжают со сдвигом, пункты появляются каскадом, нумерация, контакты. Показывается ниже 1160px; выше — обычное меню в шапке.
- [src/components/AnimatedList.tsx](src/components/AnimatedList.tsx) — список прайса на компоненте AnimatedList (React Bits), портированном на TypeScript + SCSS: анимация появления строк, градиенты по краям, свой скроллбар, навигация стрелками. Клавиатурная навигация ограничена самим списком (в оригинале слушатель висел на `window` и перехватывал Tab у всей страницы, ломая поиск).
- [src/data/concierge.ts](src/data/concierge.ts) — движок консьержа на правилах: распознаёт вопросы о вещах и пятнах на узбекской латинице, кириллице и русском, отвечает ценами из прайса. Для подключения настоящей LLM достаточно заменить `findItems`/`findStain` на вызов API — интерфейс чата не изменится.

## Карта

Leaflet + OpenStreetMap (без API-ключа). Метка ставится кликом по карте или перетаскиванием; кнопка «Моё местоположение» использует геолокацию браузера; адрес определяется автоматически через Nominatim и редактируется вручную.

## Дизайн

- Бренд: логотип клиента, переведённый в вектор из `public/logo.png` — формы и градиент измерены из исходника, не перерисованы вручную. Компоненты: [src/components/Logo.tsx](src/components/Logo.tsx) (`Logo` — знак, `LogoWord` — надпись, `LogoLockup` — полный локап с тэглайном), данные контуров: [src/data/logoPaths.ts](src/data/logoPaths.ts), скрипты трассировки: [tools/logo](tools/logo/README.md)
- Палитра: глубокий петролевый `#0B2E33`, ледяной белый `#F2F7F6`, шампань-золото `#D3AB5C`
- Шрифты (самохостинг через `@fontsource`): Playfair Display (заголовки, кириллица поддерживается), Inter (текст), JetBrains Mono (цифры/цены)
- Светлая и тёмная темы: переключатель в шапке, автоопределение по `prefers-color-scheme`, без «мигания» при загрузке
- Фирменный элемент — «швейные бирки» с символами ухода за одеждой в заголовках секций
- Адаптив до 360px, поддержка `prefers-reduced-motion`, клавиатурная навигация
