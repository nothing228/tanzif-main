# Трассировка логотипа

Скрипты, которыми `public/logo.png` (исходник от клиента) переведён в вектор.
В приложение попадает результат — [src/data/logoPaths.ts](../../src/data/logoPaths.ts); сами скрипты в сборку не входят.

Запускать только при замене исходника логотипа. Нужны `sharp` и `potrace`:

```bash
npm i -D sharp potrace
node tools/logo/analyze.cjs    # границы артворка и цвета
node tools/logo/comps.cjs      # связные компоненты (знак / надпись / тэглайн)
node tools/logo/split.cjs      # три маски по компонентам -> mask-*.png
node tools/logo/trace3.cjs     # обводка -> logo-paths.json
node tools/logo/fit.cjs        # ось и стопы градиента (МНК)
node tools/logo/build-svg.cjs  # public/logo-{mark,word,full}.svg
node tools/logo/compare2.cjs   # сверка с оригиналом (силуэт + средний цвет)
```

Пути в скриптах указаны от корня проекта.

## Как получен текущий результат

- **Кроп артворка:** x 1330, y 850, 3247×4207 (фон в PNG прозрачный, обводка — по альфа-каналу).
- **Разделение:** 38 связных компонентов → знак (4: буква + три искры), надпись «tanzif» (6), тэглайн (28).
  Резать по горизонтали нельзя: выносные элементы «t» и «f» заходят под знак.
- **Градиент:** ось измерена методом наименьших квадратов — 101,6° (почти вертикально), 5 стопов
  от `#e5c58e` до `#e8ca98`. Стопы сдвинуты на +3/+3/+6, чтобы средний цвет рендера совпал с PNG.
- **Сверка:** силуэт совпадает на 99,52% (расхождение 0,5–0,8% — субпиксельные края),
  средний цвет: оригинал `rgb(231,202,151)` против вектора `rgb(231,201,151)`.
